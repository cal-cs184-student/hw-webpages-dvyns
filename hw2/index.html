Section I: Bezier Curves and Surfaces

Part 1: Bezier Curves with 1D de Casteljau Subdivision

de Casteljau’s algorithm is a recursive geometric method for evaluating a point on a Bézier curve at a parameter t ∈ [0,1]. It works by repeatedly performing linear interpolation between consecutive control points.

The algorithm begins with the original control points. We linearly interpolate between each adjacent pair using the parameter t, producing a new smaller set of points. The interpolation process is then repeated on this new set until only a single point remains. This final point lies on the Bézier curve at parameter t.

For example, for a cubic Bézier curve with four control points:

  • Interpolate between the 4 points → produces 3 points
  • Interpolate between those 3 → produces 2 points
  • Interpolate between those 2 → produces 1 final point on the curve

In my implementation of de Casteljau's algorithm, the evaluateStep(...) method iterates over consecutive pairs of input points, computes the linear interpolation using the member variable t, and stores each result in a new vector which is returned. Each call to the function performs exactly one level of subdivision.

Part 4: Edge flip

Implementation

To implement the edge flip operation, I followed a systematic reassignment strategy. Given two triangles sharing the edge (b, c) faces (a, b, c) and (c, b, d) flipping replaces the shared edge with (a, d), producing the new faces (a, d, c) and (a, b, d).

My implementation followed four structured steps:

  1. Collect all existing mesh elements.
    Before modifying the mesh, I gathered all elements involved in the flip:
    • 10 halfedges
    • 4 vertices
    • 5 edges
    • 2 faces
    Naming each element explicitly allowed every pointer to be clearly traced and reassigned.

  2. Reassign all halfedge pointers.
    Each halfedge's next, twin, vertex, edge, and face pointers were updated to reflect the flipped configuration.
    • Face 0: (a → d → c) → h0(a→d), h8(d→c), h2(c→a)
    • Face 1: (a → b → d) → h1, h3, h4

  3. Update vertex halfedge pointers.
    Each of the four vertices was assigned a halfedge that still originates from that vertex in the updated mesh.

  4. Update edge and face halfedge pointers.
    Each edge and face was assigned one of its associated halfedges in the new configuration.

Debugging

The main issue I encountered was an incorrect vertex assignment. One vertex was pointing to a halfedge that no longer originated from it after the flip.

Because my implementation explicitly named every mesh element and separated the reassignment process into clear steps, I was able to quickly debug the problem by tracing the halfedge relationships and correcting the pointer.

Part 5: Edge split

To implement the edge split operation, I extended the same systematic pointer-reassignment approach used in Part 4. Given two triangles sharing edge (b, c) faces (a, b, c) and (c, b, d) splitting inserts a new midpoint vertex m and connects it to the opposing vertices a and d. This produces four triangles:

  • (a, b, m)
  • (a, m, c)
  • (m, b, d)
  • (m, d, c)

My implementation followed the steps below.

  1. Check for boundary edges.
    If either neighboring face is a boundary loop, the function immediately returns without modifying the mesh.

  2. Collect all existing elements.
    Before modifying any pointers, I gathered all original mesh elements:
    • 10 halfedges
    • 4 vertices
    • 5 edges
    • 2 faces
    Explicitly naming each element ensured that all pointer updates could be tracked clearly.

  3. Allocate new elements.
    An edge split introduces the following new mesh components:
    • 1 new vertex (m)
    • 3 new edges
    • 6 new halfedges (3 twin pairs)
    • 2 new faces (since the original 2 faces become 4)

  4. Compute the new vertex position.
    	v2->position = (b->position + c->position) / 2.0;
    This places the new vertex at the midpoint of the original edge.

  5. Reassign all pointers.
    All halfedge, vertex, edge, and face pointers for both old and newly created elements — were updated in a single explicit pass to reflect the new mesh structure.

Debugging

As in Part 4, the structured approach of explicitly naming every mesh element and performing pointer assignments in a single organized step made debugging straightforward.

The main challenge was ensuring that the new vertex v2 and the original vertices still pointed to valid outgoing halfedges after the split. Because some halfedges are reassigned to new faces during the operation, their corresponding vertex pointers must also be updated to maintain consistency in the halfedge structure.
